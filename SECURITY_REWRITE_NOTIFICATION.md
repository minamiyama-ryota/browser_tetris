重要: 履歴書き換え後の再クローンと手順

このリポジトリでは、デバッグアーティファクトに機密情報が含まれている可能性が確認されたため、`debug-archive*` を履歴から削除する目的で履歴書き換え（git filter-repo 等）を実行しました。影響を受ける可能性のある開発者は、ローカルの参照をリセットして古い参照による誤動作やセキュリティリスクを避けてください。

実施したこと（要約）:
- `debug-archive*` を履歴から除去（バックアップブランチを作成済）
- `JWT_SECRET` をローテーションし、GitHub Secrets を更新済

やること（必須、各自実行してください）:
1) 現在の `main` をリモートに合わせてリセット

```bash
git fetch origin
git checkout main
git reset --hard origin/main
git remote prune origin
```

2) ローカルに古いブランチがある場合は注意して処理してください。安全策としては不要なローカルブランチを削除し、必要なブランチは再作成してください。

3) 履歴の古い参照を完全に除去したい場合（注意: 破壊的操作）:

```bash
# ローカル reflog をクリアしてガベージコレクション
git reflog expire --expire=now --all
git gc --prune=now --aggressive
```

注意事項:
- 上記の `git reset --hard` や `git branch -D` 等はローカル変更を破壊します。必要な変更がある場合は事前にバックアップしてください。
- もし CI の古いランナーや外部の一時ストレージにアーティファクトが残っている可能性がある場合は、該当サービスの管理者に連絡してください。
