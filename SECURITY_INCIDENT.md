# Security Incident: debug-archive exposure (2026-04-02)

## 概要
2026-04-02 に、CI デバッグ出力をまとめた `debug-archive*` ディレクトリに JWT トークンや関連する診断ログが含まれていることを検出しました。影響を最小化するためにリポジトリ履歴から該当アーカイブを削除し、追加の対応を行いました。

## 発見日時
- 2026-04-02

## 影響
- GitHub Actions により生成されたデバッグアーカイブに `token.txt`（JWT）および `gen_debug.txt`/`verify_debug.txt` が含まれていました。これらにより署名鍵（`JWT_SECRET`）や発行済トークンの機密性が懸念されます。

## 実施済み対応
- `debug-archive*` を作業ツリーから削除し、`.gitignore` に追加。
- `git-filter-repo` を使用してリポジトリ履歴から `debug-archive*` を除去。
- コミットメッセージ中の `debug-archive*` 言及も置換して再書き込み。
- 履歴書き換えのバックアップブランチを作成・プッシュ: `backup/prune-debug-20260402-205627`。
- リポジトリの `JWT_SECRET` を新しい値へローテーションし、GitHub Actions Secrets を更新。
- セキュリティ Issue (#12) を作成し、経緯と推奨対応を通知済み。

## 残作業 / 推奨アクション
1. デプロイ先（ステージング/本番）に新しい `JWT_SECRET` を反映して、古いトークンを失効してください。トークン無効化が可能な場合は全トークンを取り消してください。
2. 全参加者にリポジトリを再クローンしてもらう（履歴書き換え済のため）。手順: `git fetch --all && git reset --hard origin/main` ではなく、ローカル変更がある場合は新規クローンを推奨します。
3. 監査ログを確認して不審なアクセスや認証試行をチェックしてください。
4. 他に漏洩の可能性があるシークレットがあればローテーションしてください（APIキー、クラウド鍵等）。
5. (任意) 追加のシークレットスキャン（gitleaks 等）を定期化するか、CI に統合してください。
